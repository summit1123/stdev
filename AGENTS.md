# AGENTS.md

## Project Goal

이 저장소의 목표는 아이의 손글씨 일기 이미지나 음성 기록을 받아
"오늘의 마음 -> 질문 씨앗 -> 미니 실험 -> 내일의 관찰" 흐름으로 바꾸는
아동 친화형 과학 탐구 서비스 MVP를 구현하는 것입니다.

## Product Promise

- 일기를 채점하지 않습니다.
- 감정을 과학적 질문의 출발점으로 바꿉니다.
- 결과는 긴 글이 아니라 카드형 콘텐츠 4장으로 제공합니다.
- 내일 다시 돌아오게 만드는 실험 미션 루프를 포함합니다.

## MVP Scope

반드시 구현할 것:

1. 일기 이미지 업로드
2. 추출 텍스트 확인 및 수정
3. 질문 카드 최대 3개
4. 실험 카드 1개
5. OpenAI TTS 내레이션
6. 결과 저장 및 다시 보기

스트레치:

1. 음성 일기 업로드
2. Sora 기반 짧은 설명 영상
3. 과학 게임형 콘텐츠
4. 상황 이미지 생성
5. 보호자 노트

## Absolute Rules

- 심리 진단, 성격 판정, 낙인 표현 금지
- 결과는 초등 고학년도 이해할 수 있는 짧은 문장으로 작성
- 질문은 최대 3개, 실험은 1개만 제공
- 실험은 가정/교실에서 안전하게 수행 가능해야 함
- OpenAI API 호출은 서버에서만 수행
- 실시간 추론 실패 시 샘플/fallback 결과로 데모가 이어져야 함
- 첫 화면은 실제 작업 도구 화면이어야 하며 마케팅 히어로처럼 보이면 안 됨

## AI Routing

- 이미지/텍스트 구조화: `gpt-5.4-mini`
- 교육 콘텐츠 정제: `gpt-5.4-mini`
- TTS: `gpt-4o-mini-tts`
- 음성 STT: `gpt-4o-mini-transcribe`
- 장면 이미지: `gpt-image-1`
- 영상 스트레치: `sora-2-pro`
- moderation: `omni-moderation-latest`

## UX Rules

- 첫 화면에서 바로 업로드나 샘플 시작이 가능해야 함
- 업로드, 분석중, 완료, 실패 상태를 모두 노출
- 결과의 핵심 가치는 상황 이미지, 과학 해석, 게임형 콘텐츠, 미션 루프에 집중
- 오디오 재생은 미디어 결과와 같은 캔버스에서 바로 보이게 둘 것
- 상태, 시나리오, 미션, 다시 보기 히스토리는 분리된 inspector에서 스크롤 가능해야 함

## Delivery Bar

작업은 아래가 충족되어야 완료입니다.

1. 웹에서 실제 업로드 -> OCR 수정 -> 분석 -> 영상/오디오/결과 -> 저장 흐름이 동작함
2. 샘플 일기 3종 이상이 안정적으로 재생 가능함
3. 결과 데이터가 명시적 스키마를 가짐
4. 오디오 생성 실패 시에도 결과 카드가 유지됨
5. 영상 생성 실패 시에도 상황 이미지와 카드 결과가 유지됨
6. lint/build/backend smoke test가 통과함
