# Mode Contract: Implementation

Use this mode when the goal is real code, tests, runtime behavior, or a production-minded feature slice.

## Source of truth

- the codebase itself
- tests and local checks
- `.codex-loop/prd/PRD.md`
- `.codex-loop/tasks.json`

## Completion bar

- The changed path works end to end.
- Evidence exists in tests, checks, screenshots, or logs.
- Supporting contracts and docs match the repo state.

## Fail when

- Completion is claimed on scaffolding alone.
- Checks were skipped or weak.
- The UI or workflow still feels generic or unfinished.
