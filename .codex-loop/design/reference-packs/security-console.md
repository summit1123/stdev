# Reference Pack: security-console

Use this when the product is an operational security tool, verification console, risk dashboard, or evidence-heavy control surface.

## Core feel

- calm, exact, and accountable
- stable panes, logs, evidence tables, inspectors, and timelines
- color used sparingly for status, not decoration
- every view should help an operator make or justify a decision

## Structural cues

- prefer split panes, stacked log regions, and deliberate comparison tables
- keep provenance, rule hits, severity, and timestamps visible
- show before/after, blocked/allowed, or pass/fail states side by side when useful
- use screenshots only when they reveal real runtime state

## Good fits

- red-team / blue-team orchestration tools
- guardrail tuning consoles
- audit and evidence products
- release gates for AI workflows

## Avoid

- large hero headers inside the tool
- analytics confetti with no action attached
- colorful status pills everywhere
- soft marketing cards carrying operational information
