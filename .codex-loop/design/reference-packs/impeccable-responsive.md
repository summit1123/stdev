# Reference Pack: impeccable-responsive

Inspired by `impeccable`, use this when the product has to hold together cleanly across desktop, tablet, and mobile while preserving hierarchy and task flow.

## Core feel

- flexible without feeling loose
- layout decisions are intentional at every breakpoint
- interaction density adjusts with purpose rather than simple shrinking
- responsive behavior is part of the product design, not an afterthought

## Structural cues

- use container-aware layout regions instead of one rigid desktop composition
- keep navigation, toolbars, and inspectors stable across breakpoints where possible
- let tables degrade deliberately into stacked or segmented views
- define responsive states for empty, loading, error, and success paths
- preserve the main action path before secondary utility

## Anti-patterns to avoid

- desktop screenshots treated as proof of complete design quality
- cramped touch targets and collapsed type scales on mobile
- unplanned wrapping that breaks hierarchy or action grouping
- switching to entirely different visual logic between breakpoints

## Good fits

- operational web apps used across devices
- public-service and field workflows
- approval or review tools with mobile follow-through
- products that combine dense desktop views with lightweight mobile actions
