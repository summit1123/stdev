# Reference Pack: devtool-minimal

Use this when the product is a builder tool, engineering dashboard, or internal utility that should feel sharp and lightweight without looking unfinished.

## Core feel

- direct, compact, and engineering-first
- visual discipline over flourish
- table, log, and code surfaces are first-class
- interactions should feel deterministic

## Structural cues

- keep navigation tight and predictable
- make command output, logs, diffs, and states easy to scan
- let spacing and border rhythm do most of the grouping work
- use accent color only to orient or warn

## Good fits

- CI dashboards
- harness and agent control panels
- developer-facing review tools
- configuration editors

## Avoid

- onboarding-style empty states in the main workflow
- novelty illustration systems
- split marketing layouts
- oversized iconography standing in for real information
